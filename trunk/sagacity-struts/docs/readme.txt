docs目录是整个项目的文档目录，其按照项目的周期和项目过程管理的特点设置相关目录

项目开发类文档
business：商务阶段，包括商务售前的文档和合同信息
requirement:需求阶段,包括需求文档，需求用例，需求变更等文档
analysis&design:分析设计阶段
architecture:架构阶段
test:测试阶段,包括测试用例，测试报告等
deliver:交付阶段：一般包括最终上线的部署程序包，项目源码，系统架构文档，需求文档，
数据库设计文档，应用程序设计文档，用户操作手册，运维手册，测试报告，上线报告等文档

管理类文档
management：项目管理文档目录,包括会议纪要，项目考勤，项目周报等
schedule:项目计划，一般存放mpp之类的文件，跟踪团队成员的项目进度
specification:规范，针对项目指定的一些规范，包括数据库设计规范，编码规范等等
